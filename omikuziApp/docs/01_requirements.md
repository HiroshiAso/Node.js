# 要件定義書

## 1. はじめに
本書は、おみくじアプリ（Omikuzi App）の要件定義をまとめたものである。
本システムは、Webブラウザ上で動作するシンプルなおみくじアプリケーションであり、ユーザーに対して手軽な占い体験を提供することを目的とする。また、開発者にとってはNode.jsおよびExpressフレームワークの学習用教材としての側面も持つ。

## 2. システム概要
本システムは、ユーザーがブラウザからアクセスすることで、ランダムに選出された運勢（大吉、中吉など）を表示するWebアプリケーションである。

## 3. 業務要件
### 3.1 ターゲットユーザー
- 一般Webユーザー
- Node.js/Express学習者

### 3.2 利用シーン
- ユーザーが運試しをしたい時
- 隙間時間の暇つぶし

## 4. 機能要件

| ID | 機能名称 | 概要 | 備考 |
| :--- | :--- | :--- | :--- |
| FR-01 | おみくじ実行機能 | トップページへのアクセス時に、サーバー側で運勢をランダムに決定する。 | 運勢は「大吉」「中吉」「小吉」「吉」「凶」の5種類。 |
| FR-02 | 結果表示機能 | 決定された運勢を画面上に大きく表示する。 | |
| FR-03 | 再実行機能 | 画面上の操作により、再度おみくじを引くことができる（ページリロード）。 | 「もう一度引く」ボタン等を設置。 |

## 5. 非機能要件

### 5.1 ユーザビリティ
- シンプルで直感的なUIであること。
- 結果が一目でわかる視認性を確保すること。

### 5.2 可用性
- 特段のSLAは設けないが、学習用サーバー起動中は常時アクセス可能であること。

### 5.3 性能
- リクエストからレスポンスまでの時間が、ユーザーにストレスを与えない範囲（数秒以内）であること。

### 5.4 拡張性
- 将来的に運勢の種類を増やしたり、確率を変更したりすることが容易なコード構造であること。

## 6. 技術制約
- **ランタイム**: Node.js
- **フレームワーク**: Express
- **テンプレートエンジン**: EJS
- **フロントエンド**: HTML5, CSS3
