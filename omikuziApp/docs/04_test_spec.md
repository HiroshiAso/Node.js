# テスト仕様書

## 1. テスト方針
本アプリケーションは小規模であるため、手動によるブラックボックステスト（結合テストレベル）を中心に実施する。
重点確認項目は以下の通り。
1.  アプリケーションが正常に起動し、アクセス可能であること。
2.  おみくじの結果がランダムに表示されること。
3.  再実行が可能であること。

## 2. テスト環境
-   **OS**: 開発機（Windows/Mac）または Dockerコンテナ
-   **ブラウザ**: Google Chrome (最新版)
-   **Node.js**: v14以上

## 3. テストケース一覧

| テストID | 確認項目 | 前提条件 | 操作手順 | 期待値 | 結果 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| T-01 | 起動確認 | `npm start` でサーバー起動済み | ブラウザで `http://localhost:3000` にアクセスする。 | 画面が表示され、エラーとならないこと。<br>タイトル「おみくじアプリ」が表示されること。 | |
| T-02 | 運勢表示確認 | T-01 実施後 | 画面中央の運勢表示エリアを確認する。 | 「大吉」「中吉」「小吉」「吉」「凶」のいずれかが表示されていること。 | |
| T-03 | 再実行確認 | T-02 実施後 | 「もう一度引く」ボタンをクリックする。 | 画面がリロードされ、再度運勢が表示されること。（前回と異なる結果になる場合もある） | |
| T-04 | ランダム性確認 | T-03 実施後 | 「もう一度引く」を10回程度繰り返す。 | 常に同じ結果ではなく、異なる運勢が表示されること。 | |
| T-05 | 存在しないパス | サーバー起動済み | ブラウザで `http://localhost:3000/dummy` にアクセスする。 | 404 Not Found エラー画面が表示されること。 | |

## 4. エビデンス
テスト実施時は、各テストケースの結果（OK/NG）を記録し、必要に応じて画面キャプチャを取得する。
